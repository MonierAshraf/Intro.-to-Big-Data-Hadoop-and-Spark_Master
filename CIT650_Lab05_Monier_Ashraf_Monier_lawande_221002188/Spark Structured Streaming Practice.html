Introduction:¶

Batch vs Stream data processing:¶

Batch data processing is the processing of data as a collection of points that have been stored or grouped within a specific time interval. While in stream data processing, the data is fed into the analytics system as soon as the data arrive.

Apache Spark Structured Streaming:¶

Spark Structured Streaming is an open source scalable and fault-tolerant stream processing engine built on the top of Spark SQL engine. Using Spark Structured Streaming, you can express streaming data processing as batch.

In this lab, we are going to practice SQL-like operations (e.g. Select, Where, GroupBy)

Programming Model:¶

Structured Streaming treats a data stream as a table that is being continuously appended. This is what makes stream processing in Structured Streaming similar to batch processing.

A query on the input data generates the Results Table.

Output Modes:¶

By output, we mean that the data written to the sink and there are three modes for output:

1. Complete Mode - The whole updated Results Table will be written to the sink.
2. Append Mode - Only the appended part in the Results Table, since the last write operation, will be written to the sink. Obviously, it is applicable when the rows in the Results Table are not expected to change.
3. Update Mode - Only the rows that were updated in the Results Table, since the last write operation, will be written to the sink.

Please refer to this link for more information about Spark Structured Streaming

from pyspark.sql import SparkSession
from pyspark.sql.functions import explode
from pyspark.sql.functions import split


# build spark session
sc = SparkContext()
spark = SparkSession(sc) \
    .builder \
    .appName("Spark Structured Streaming Lab") \
    .getOrCreate()

# Set the log level to error
spark.sparkContext.setLogLevel("ERROR")

#start reading from the socket on port 5555
lines = spark.readStream.format("socket").option("host","localhost").option("port",5555).load()

# Split lines
words = lines.select(explode(split(lines.value," ")).alias("word"))

# count words
wordCounts = words.groupBy("word").count()

# write the stream
query = wordCounts.writeStream.outputMode("complete").format("console").start()
query.awaitTermination()

from pyspark.sql import SparkSession
from pyspark.sql.functions import explode
from pyspark.sql.functions import split
from pyspark.sql.functions import *


spark = SparkSession.builder.appName("Spark Structured Streaming Lab").getOrCreate()
spark.sparkContext.setLogLevel("ERROR")

# Added includeTimestamp option 
lines = spark.readStream.format("socket").option("host","localhost").\
option("port",5553).option("includeTimestamp", "true").load()

# Split line into => word, Time 
words = lines.select(explode(split(lines.value, ' ')).alias('word'),lines.timestamp)

# Group by Timestamp, word
The_window_count = words.groupBy(window(words.timestamp, '10 seconds', '5 seconds'),words.word).count()

# Stream the output
query = The_window_count.writeStream.outputMode("complete").format("console").start()
query.awaitTermination()
